# 江浙沪旅游行程规划系统重构 - 功能需求文档

## 1. 项目概述

### 1.1 项目背景
江浙沪旅游行程规划系统是一款面向江浙沪地区旅游的智能行程规划工具。本次重构聚焦于"创建行程"和"我的行程"两大核心模块，实现数据共通功能，优化用户体验，并确保行程导出功能与首页各模块数据互通。

### 1.2 重构目标
- 实现"创建行程"和"我的行程"模块的数据共通
- 优化用户交互体验，简化操作流程
- 提供多格式行程导出功能
- 确保首页数据与各模块实时同步
- 建立统一的设计语言和交互规范

## 2. 功能需求分析

### 2.1 创建行程模块

#### 2.1.1 核心功能
| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 分步引导创建 | 采用4步向导式流程：基本信息→目的地选择→兴趣偏好→预算规划 | 高 |
| 目的地多选 | 支持最多选择3个目的地，实时显示已选目的地预览 | 高 |
| 兴趣偏好选择 | 8种兴趣标签多选，包括自然风光、历史文化、美食购物等 | 高 |
| 同行人员选择 | 5种类型单选：情侣、亲子、独行、朋友、家庭 | 高 |
| 旅游风格选择 | 4种风格单选：精品深度、高效紧凑、休闲放松、丰富多样 | 高 |
| 智能预算规划 | 滑块调节总预算，自动分配各分类预算并可视化展示 | 高 |
| 表单验证 | 每步骤必填项验证，错误提示引导 | 高 |

#### 2.1.2 用户操作流程
```
开始创建 → 填写基本信息 → 选择目的地 → 设置偏好 → 预算规划 → 生成行程
```

### 2.2 我的行程模块

#### 2.2.1 核心功能
| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 行程列表展示 | 支持网格/列表双视图切换 | 高 |
| 行程搜索 | 按标题、目的地关键词搜索 | 高 |
| 状态筛选 | 按行程状态筛选：规划中/已确认/进行中/已完成 | 高 |
| 排序功能 | 支持按创建时间、预算、天数排序 | 中 |
| 标签页切换 | 全部/进行中/已完成快速切换 | 高 |
| 行程操作 | 查看、分享、导出、删除 | 高 |
| 统计展示 | 实时显示行程数量统计 | 中 |

#### 2.2.2 行程状态管理
- **规划中**：刚创建的行程，可编辑修改
- **已确认**：行程已确认，准备出行
- **进行中**：当前日期在行程日期范围内
- **已完成**：行程结束日期已过

### 2.3 数据共通功能

#### 2.3.1 数据同步范围
| 数据类型 | 同步方向 | 说明 |
|----------|----------|------|
| 行程数据 | 创建行程 → 我的行程/首页 | 创建后立即同步 |
| 预算数据 | 创建行程 → 首页统计 | 实时更新总预算统计 |
| 目的地数据 | 创建行程 → 首页统计 | 去重统计目的地数量 |
| 用户偏好 | 创建行程 → 首页配置 | 保存用户常用偏好设置 |

#### 2.3.2 数据存储方案
```javascript
// 行程数据结构
{
  id: Number,              // 唯一标识
  title: String,           // 行程标题
  departure: String,       // 出发地
  startDate: String,       // 开始日期
  endDate: String,         // 结束日期
  days: Number,            // 行程天数
  destinations: Array,     // 目的地数组
  interests: Array,        // 兴趣偏好数组
  companion: String,       // 同行人员
  travelStyle: String,     // 旅游风格
  budget: Number,          // 总预算
  budgetBreakdown: Object, // 预算明细
  status: String,          // 行程状态
  createdAt: String        // 创建时间
}
```

### 2.4 行程导出功能
